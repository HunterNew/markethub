Admin Workflows

Vendor Approval Flow

  1. Vendor registers at /auth/register/vendor → status: "pending"
  2. Admin sees in Vendors page → clicks Approve/Reject
  3. If approved → vendor can add products
  4. Admin can later Disable vendor → all products hidden from storefront

Return & Refund Flow

  1. Customer requests return (with proof image)
  2. Vendor approves → status becomes "refund_pending"
  3. Admin sees in Returns page → clicks "Process Refund"
  4. Order status → "Returned", payment → "Refunded"

🔧 Troubleshooting & Common Issues

❌ "ECONNREFUSED" errors

Backend server not running. Start: cd backend && npm run dev

❌ Email OTP not sending

  • Check backend/.env has correct SMTP_USER and SMTP_PASS
  • For Gmail: use App Password (not regular password)
  • Restart backend after changing .env

❌ OTP verification fails

  • Send a NEW OTP (old ones expire in 10 min)
  • Check MySQL timezone matches system

❌ Migration fails "Duplicate column"

  • Delete the migration record: DELETE FROM migrations WHERE filename='xxx.sql'
  • Re-run: npx ts-node src/db/migrate.ts

❌ Vendor disabled but products still showing

  • Restart backend after disabling vendor
  • Products filter by v.status = 'approved' — disabled vendors are automatically hidden

❌ MRP not showing on product

  • MRP must be greater than Retail Price to show strikethrough
  • MySQL returns decimals as strings — ensure Number() comparison

❌ Theme not changing

  • Go to Admin → Settings → Storefront Theme
  • Select theme and refresh the storefront page
  • Check /api/v1/settings/theme returns correct value

Vendor Workflows

Adding a Product

  1. Go to Products → Add Product
  2. Fill: Name, Description, MRP, Retail Price, Stock, Category, Brand
  3. Upload images (up to 5) — URL or file upload
  4. Set delivery charge type (per piece, per kg, or store default)
  5. Optionally enable wholesale pricing (price + min qty)
  6. Optionally add variants (size, color, etc.) with individual prices/stock
  7. Submit → status: "pending_approval" → admin approves

Product Pricing Display

  1. MRP shows as strikethrough with % off badge
  2. Retail Price is the selling price (what customer pays)
  3. If offer active: Retail Price shows as strikethrough, offer price displayed
  4. If wholesale enabled: wholesale price shown for bulk buyers

Enable/Disable Products

  1. Go to Products list
  2. Click eye icon to disable (product hidden from storefront)
  3. Click again to re-enable

CSV Bulk Import

  1. Go to Products → Import
  2. Upload product images first (bulk upload)
  3. Paste CSV data with columns: name, description, price, category_name, stock
  4. Optional: image_url, wholesale_enabled, wholesale_price, wholesale_min_qty
  5. Tamil text supported in name and description fields

Customer Workflows

Registration (Separate Pages)

  1. Customer: /auth/register — simple form (name, email, phone, password)
  2. Seller: /auth/register/vendor — includes store name, GST, bank details
  3. Email OTP verification required for both

Placing an Order

  1. Browse products → Add to cart (or from wishlist)
  2. Go to Cart → Apply coupon (optional) → Checkout
  3. Select/add delivery address
  4. Choose payment: COD (if vendor allows) or Razorpay
  5. Confirm & Pay → Order confirmation shown
