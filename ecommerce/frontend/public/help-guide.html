Admin Workflows

Vendor Approval Flow

  1. Vendor registers → status: "pending"
  2. Admin sees in Vendors page → clicks Approve/Reject
  3. If approved → vendor can add products
  4. If rejected → reason shown to vendor

Return & Refund Flow

  1. Customer requests return (with proof image)
  2. Vendor approves → status becomes "refund_pending"
  3. Admin sees in Returns page → clicks "Process Refund"
  4. Order status → "Returned", payment → "Refunded"
  5. Customer notified: "Amount credited in 3-5 days"

Withdrawal Flow

  1. Vendor requests withdrawal from earnings
  2. Admin sees in Withdrawals → Approve/Reject
  3. If approved → vendor balance deducted

Vendor Workflows

Adding a Product

  1. Go to Products → Add Product
  2. Fill name, description, price, category, stock
  3. Upload images (up to 5)
  4. Optionally enable wholesale pricing
  5. Optionally add variants (size, color, etc.)
  6. Submit → status: "pending_approval" → admin approves

Creating a Sale Offer

  1. Go to Offers → Single Offer or Bulk % Off
  2. Select product(s), set discount percentage
  3. Set start and end date
  4. Sale price auto-calculated and shown to customers

Order Fulfillment

  1. New order appears → status: "Confirmed"
  2. Print invoice → paste on package
  3. Click "Mark Shipped" → customer notified
  4. After delivery → Click "Mark Delivered"
  5. Commission calculated, earnings updated

Invoice Details

  1. Shows: Sold By (your address, GST, FSSAI)
  2. Billing & Shipping address
  3. Items table with tax (CGST/SGST)
  4. Amount in words, authorized signatory (your logo/signature)

Customer Workflows

Placing an Order

  1. Browse products → Add to cart
  2. Go to Cart → Apply coupon (optional) → Checkout
  3. Select/add delivery address
  4. Choose payment: COD (if vendor allows), Razorpay, or Stripe
  5. Review order → Confirm & Pay
  6. Order confirmation page shown

Return & Refund

  1. Go to order detail (delivered order)
  2. Click "Request Return" (only if vendor has return policy enabled)
  3. Upload proof image + write reason
  4. Vendor reviews → approves/rejects
  5. If approved → admin processes refund
  6. Amount credited in 3-5 business days

Registration

  1. Enter email → Click "Send OTP" → verify email
  2. Enter phone number (required)
  3. Set password → Create Account

🔧 Troubleshooting & Common Issues

❌ "ECONNREFUSED" errors in frontend

Backend server is not running. Start it:

cd backend && npm run dev

❌ Email OTP not sending

  • Check backend/.env has correct SMTP_USER and SMTP_PASS
  • For Gmail: use App Password (not regular password)
  • Google Account → Security → 2-Step Verification → App Passwords
  • Restart backend after changing .env

❌ OTP verification fails ("Invalid or expired")

  • Send a NEW OTP (don't use old ones)
  • Check MySQL timezone matches your system
  • OTP expires in 10 minutes
  • Run: SELECT NOW(); in MySQL to verify time

❌ Migration fails with "Duplicate column"

  • Column already exists from a previous failed migration
  • Delete the migration record: DELETE FROM migrations WHERE filename='xxx.sql'
  • Remove the duplicate ALTER from the .sql file
  • Re-run: npx ts-node src/db/migrate.ts

❌ MySQL GROUP BY error

Wrap non-grouped columns with ANY_VALUE() in SELECT

❌ Firebase Phone Auth "BILLING_NOT_ENABLED"

  • Firebase Phone Auth requires Blaze plan (pay-as-you-go)
  • Free tier: 10,000 verifications/month
  • But requires a billing account (credit card) linked

❌ WhatsApp button not showing

  • Check ADMIN_WHATSAPP is set in backend/.env
  • Restart backend after adding it
  • Number format: country code + number (e.g., 919876543210)

❌ Offer price not applying in cart

  • Check offer dates (must be active: start ≤ now ≤ end)
  • Restart backend to pick up code changes
  • Remove item from cart and re-add

❌ Invoice not showing vendor details

  • Vendor must fill: Business Address, GST, Signature in Profile
  • These fields appear on invoice only if filled
